       - If mode is `'twilio'`:
         - Import twilio: `import twilio from 'twilio';`
         - Create client: `const client = twilio(accountSid, authToken)` using env vars `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`.
         - Send via: `client.messages.create({ from: 'whatsapp:' + process.env.TWILIO_WHATSAPP_FROM, body, to: 'whatsapp:' + to })`
       - If mode is `'log'` (default):
         - `console.log('\n[WhatsApp Dev] ---')`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API | Unauthenticated HTTP requests to notification endpoints (mitigated by authenticate middleware) |
| API -> Twilio | Outbound HTTPS to Twilio API containing PII (phone numbers, lot data) |
| .env -> Process | Twilio credentials loaded into memory from environment variables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | POST /api/notifications/* | mitigate | Require `authenticate` middleware on all notification endpoints — only logged-in users can trigger notifications |
| T-03-02 | Information Disclosure | server/.env, process.env | mitigate | Twilio credentials stored in .env only (never in source). .env.example uses placeholder values. No credential logging in dev mode. |
| T-03-03 | Tampering | WhatsApp message templates | mitigate | Message templates defined server-side in whatsapp.js (formatArrivalMessage, formatDispatchMessage). Client cannot control message body — only provides lotNumber, partyName, quantity. |
| T-03-04 | Denial of Service | Twilio API | accept | Dev mode (WHATSAPP_MODE=log) is default — no Twilio costs. Production mode can add rate limiting later if abuse occurs. Low priority for factory floor context. |
| T-03-SC | Tampering | twilio npm package | mitigate | Human verification checkpoint before npm install (Task 2). Verify publisher, downloads, and recency at npmjs.com/package/twilio. |
</threat_model>
