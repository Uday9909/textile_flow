<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (browser) UI | No sensitive data rendered — only form inputs and status messages |
| Client -> API /forgot-password | Email address crosses here (low sensitivity for factory tool) |
| Client -> API /reset-password | Token + new password crosses here over HTTPS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | Reset token in browser URL | accept | Dev-mode only (link logged to console). In production, token is sent via email and HTTPS protects the channel. URL history exposure is acceptable for factory tool threat model. |
| T-02-09 | Tampering | Reset token in URL could be bookmarked/shared | accept | Token is single-use with 15-min expiry. Reuse of a consumed or expired token returns 400 error (handled by Plan 01 backend). |
| T-02-10 | Information Disclosure | Forgot-password form email input | accept | Same response returned for found/not-found email (handled by Plan 01 backend prevents enumeration). |
| T-02-SC | Tampering | npm install | accept | No new npm dependencies introduced in this plan |
</threat_model>
