## Password Reset Token Lifecycle

1. Server generates crypto.randomUUID token
2. SHA-256 hash stored in password_reset_tokens table
3. Raw token returned in reset link (logged to console)
4. On reset: raw token hashed to SHA-256, looked up in db
5. If valid and not expired: password updated, token marked used=1
6. Used tokens cannot be replayed (used flag prevents double-use)
7. Expired tokens return 400 even if not used
</threat_model>
